# 回归 · 最小二乘法

> 拖动散点和直线,亲眼看"残差平方和"如何随你手中的直线变化,
> 再按一个键,看梯度下降把它滚到谷底——顺便回收「点积」demo 埋下的伏笔:
> 最小二乘的解,就是一次投影。

## 如何使用

1. 打开本目录的 [index.html](https://wanghaoming.github.io/math4llm/demos/least-squares/index.html)(或从合集[首页](https://wanghaoming.github.io/math4llm/)进入)。
2. 画布上有一批**红色散点**和一条**蓝色直线**(由两个蓝色端点定义):
   - **拖动红点**改变数据;**点击空白处**加新点(最多 15 个);**双击红点**删除它;
   - **拖动两个蓝色端点**手动调整直线,右侧面板实时显示斜率 k、截距 b 和残差平方和 SSE。
3. 三个开关:
   - **显示残差**:每个点到直线的竖直白色虚线,长度 = 这个点的"误差";
   - **显示残差的平方**:以残差为边长的黄色正方形——**面积**就是这个点对 SSE 的贡献,SSE = 所有黄色面积之和;
   - **显示最佳拟合参考线**:绿色虚线,闭式解算出的全局最优直线。
4. 两个按钮:
   - **一键拟合:梯度下降"下山"**——从你当前的直线出发,对 (k, b) 做真实的梯度下降动画,看 SSE 一路下降、直线缓缓转到最优位置;
   - **重置数据**:恢复初始散点与直线。

先手动拖直线,试着把 SSE 压到最小,再和面板里的"最小 SSE"比一比——
然后按「一键拟合」,看机器怎么下山。

## 〇、先说结论

> **最小二乘 = 在所有直线里,挑"残差平方和"最小的那条。**
> 它同时是三样东西:统计里最经典的**回归**;神经网络里**均方误差损失 +
> 梯度下降**的最简版本(一个神经元、没有激活函数);几何里的**一次投影**——
> 「点积」demo 里那只影子,在这里再次登场。

## 一、问题:一堆散点,哪条直线"最好"?

散点大致沿一条直线分布,但每个点都有噪声。任意画一条直线 y = kx + b,
每个数据点 (xᵢ, yᵢ) 和直线之间就有一段竖直的落差:

$$e_i = y_i - (k x_i + b) \qquad \text{(第 i 个点的残差:实际值减预测值)}$$

```
        y
        |                * (x2, y2)
        |                .                <- 残差 e2 > 0(点在线上方)
        |             ___.____
        |     ____----
        | ----    .
        |         .                       <- 残差 e1 < 0(点在线下方)
        |         * (x1, y1)
--------+------------------------- x
```

"最好的直线"应该让所有残差整体最小。但"整体"怎么定义?直接把残差加起来
不行——上方的正残差和下方的负残差会互相抵消,一条烂线也能得零分。
最小二乘的选择是**先平方再求和**:

$$\mathrm{SSE}(k, b) = \sum_i e_i^2 = \sum_i \big(y_i - (k x_i + b)\big)^2$$

为什么是平方,而不是绝对值?三个理由:

- **正负不抵消**:平方后全为正,谁也别想蒙混过关;
- **处处可导**:|e| 在零点有个尖角,没法求导;e² 光滑,闭式解和梯度下降
  两条路都走得通(这正是 demo 两个功能的数学前提);
- **大错重罚**:残差 2 的代价是残差 1 的 4 倍。演示里勾选"显示残差的平方",
  黄色正方形的**面积**就是各点的罚分——把一个点拖远,看它的正方形以平方
  速度膨胀,你就明白为什么直线会被离群点狠狠拽过去。

> **SSE 就是画布上所有黄色正方形的总面积。"拟合"的全部含义:
> 转动、平移那条直线,让黄色总面积最小。**

## 二、怎么求最小:两条路

SSE 是 k 和 b 的函数——一个开口朝上的"碗"(二次曲面)。找碗底有两条路,
demo 里各对应一个功能。

### 2.1 闭式解:让导数等于零(面板里的 k*、b*)

碗底的特征是"往哪个方向都不再下降",即两个偏导数同时为零。解这个方程组
(初中代数就够),得到:

$$k^* = \frac{\sum_i (x_i - \bar{x})(y_i - \bar{y})}{\sum_i (x_i - \bar{x})^2} \qquad b^* = \bar{y} - k^* \bar{x}$$

其中 x̄、ȳ 是均值。两个式子各有一句人话:

- **k\*** 的分子是 x 和 y 的"协同变化量",分母是 x 自身的"分散程度"——
  斜率 = y 随 x 变化的部分,占 x 总变化的比例;
- **b\* 的式子等价于:最优直线必过均值点 (x̄, ȳ)**。拖动散点时盯着绿色
  参考线,它始终穿过数据的"重心"。

面板里的 k*、b*、最小 SSE 就是按这两个式子实时算的。注意分母 Σ(x−x̄)²:
如果把所有点拖成一条竖线(x 全相同),分母为零,最优直线不存在——
面板会显示"—",这不是 bug,是数学在报错。

### 2.2 梯度下降:不解方程,下山(「一键拟合」按钮)

第二条路不解任何方程:站在碗壁上的当前位置 (k, b),摸一摸脚下坡度
(求 SSE 对 k、b 的偏导),朝下坡方向挪一小步,重复:

$$k \leftarrow k - \eta \frac{\partial \mathrm{SSE}}{\partial k} \qquad b \leftarrow b - \eta \frac{\partial \mathrm{SSE}}{\partial b}$$

按下「一键拟合」,你看到的就是这个过程逐帧播放:SSE 数字一路变小,
直线缓缓转向、平移,最后停在和绿色参考线重合的位置。

如果你玩过「什么是神经网络」demo,应该觉得眼熟——**这就是那 3750 步
"猜 → 对答案 → 拧旋钮"的最简版本**:

| | 什么是神经网络 | 本 demo |
| --- | --- | --- |
| 参数(旋钮) | 13,184 个 | 2 个(k 和 b) |
| 损失函数 | 交叉熵(分类) | SSE / 均方误差(回归) |
| 求解方式 | 只能梯度下降 | 闭式解、梯度下降都行 |
| 损失面 | 高维,画不出来 | 二维的碗,一眼看穿 |

**线性回归就是最简单的"神经网络"**:一个神经元、恒等激活、均方误差损失。
参数只剩两个,整个"训练"过程第一次变得肉眼可见——这正是本 demo 存在的
意义。(为什么真实网络不用闭式解?因为一过激活函数,碗就不再是碗,
方程解不动了,只剩下山这一条路。)

两条路殊途同归——闭式解直接跳到碗底,梯度下降滚到碗底,终点是同一个。
这不是巧合:第三节末尾会给出把它们焊死的那条恒等式。

## 三、几何意义:最小二乘 = 一次投影

这是「点积」demo 结尾埋的伏笔,现在回收。换一副眼镜看同一个问题:
不再把数据看成"平面上的 n 个点",而是把所有观测值**竖着堆成一个向量**:

$$\mathbf{y} = (y_1, y_2, \ldots, y_n) \qquad \text{(n 维空间里的一个箭头)}$$

直线能产生的预测也是一个 n 维向量,而且只有两个自由度:

$$\hat{\mathbf{y}} = k \cdot (x_1, \ldots, x_n) + b \cdot (1, \ldots, 1)$$

k 和 b 随便取,所有可能的预测 ŷ 张成 n 维空间里的一张**二维平面**
(模型平面)。真实观测 y 因为带噪声,一般**不在**这张平面上。
于是"找最好的直线"变成了一个纯几何问题:

> **在模型平面上,找离 y 最近的那个点。**

答案你在「点积」demo 里已经见过:**把 y 垂直投影到平面上**。

```
              * y    (观测向量, n 维空间中一点)
             /.
            / .
           /  .      <- 残差向量 e = y - yhat, 垂直于平面
          /   .
         /    .
   -----+-----+----------------
        O     yhat   (y 在模型平面上的投影 = 最小二乘的预测)
```

"最近" = 残差向量 e 最短,而 SSE 恰好就是 |e|²(平方和 = 长度的平方,
「点积」demo 3.4 的副产品 a·a = |a|²)。垂直投影最短,所以:

> **最小二乘的解 = y 在模型平面上的投影;残差向量 ⊥ 模型平面。**

"垂直"翻译成代数,就是两个点积为零(e 和平面的两个方向都零贡献——
「点积」demo 2.2 那个"垂直 = 互不贡献"):

$$\mathbf{e} \cdot (x_1, \ldots, x_n) = 0 \qquad \mathbf{e} \cdot (1, \ldots, 1) = 0$$

把这两个点积展开,得到的正是 2.1 节的闭式解——教科书管它叫**正规方程**。

### 下山与投影:同一组方程的两种读法

现在兑现 2.2 节的承诺:**"最小化 MSE"(第二节)和"投影"(本节)不是两个
话题,而是同一个最小化问题的两副面孔——MSE 是"要最小化的东西",投影是
"最小化完成后答案的几何形状"。**

**先看清 MSE 生来就是几何量。** 上面已经说了 SSE = |e|²,换个写法:

$$\mathrm{SSE} = \sum_i (y_i - \hat{y}_i)^2 = |\mathbf{y} - \hat{\mathbf{y}}|^2 \qquad \text{(预测向量到观测向量的距离的平方)}$$

所以"最小化 MSE"从第一个字起就是几何语言:**在模型平面上滑动 ŷ,
让它到 y 的距离最短**。损失函数不是"碰巧能几何解释",它的定义就是距离。

**训练时其实有两个空间在同步演化**,初学最容易混的就是这里:

| | 参数空间(2.2 的舞台) | 观测空间(本节的舞台) |
| --- | --- | --- |
| 住着谁 | 参数 (k, b),只有 2 维 | 向量 y、ŷ,n 维 |
| 损失长什么样 | 一只碗:SSE(k, b) 曲面 | 一段距离:y 到模型平面多远 |
| 训练在干什么 | 沿碗壁**下山** | ŷ 在模型平面上**滑动**,靠近 y |
| 终点 | 碗底(梯度 = 0) | 垂足(y 的投影) |

关键在于这两幅画面**逐帧对应**:每组 (k, b) 唯一对应平面上的一个 ŷ。
你按「一键拟合」时,面板里 (k, b) 每挪一步(参数空间下山),背后那个
n 维的 ŷ 就在模型平面上滑一步、离 y 更近一点。**碗底和垂足,
是同一个时刻的两张照片。**

**最后是焊死它们的恒等式。** 对 SSE 求偏导(记残差 eᵢ = yᵢ − kxᵢ − b):

$$\frac{\partial\,\mathrm{SSE}}{\partial k} = -2\sum_i x_i e_i = -2\,(\mathbf{e} \cdot \mathbf{x}) \qquad \frac{\partial\,\mathrm{SSE}}{\partial b} = -2\sum_i e_i = -2\,(\mathbf{e} \cdot \mathbf{1})$$

看清楚右边:**SSE 的两个梯度分量,恰好就是"残差与模型平面两个方向的
点积"**(乘个 −2)。不是相似,不是类比,是同一个式子。于是:

$$\text{梯度} = 0 \iff \mathbf{e}\cdot\mathbf{x} = 0 \ \text{且}\ \mathbf{e}\cdot\mathbf{1} = 0 \iff \text{残差} \perp \text{模型平面} \iff \text{垂足}$$

"下山下到底"和"垂直投影"是**同一组方程的两种读法**:用微积分的嘴念出来
叫"驻点条件",用几何的嘴念出来叫"正规方程"。「点积」demo 里
"垂直 = 点积为零 = 零贡献"的直觉,在这里成了训练收敛的判据——
残差里再也榨不出任何模型方向上的成分,梯度自然为零:

> **学无可学之时,恰是残差与模型互相垂直之刻。**

**到真正的神经网络,哪些留下、哪些变了?** 留下的:回归网络的 MSE 依旧是
|y − ŷ|²,训练依旧是"在可达预测的集合里找离 y 最近的点",收敛时残差依旧
垂直于这个集合(的切方向)。变了的:线性模型可达的预测是一张**平的**平面,
垂足有闭式解;套上激活函数后它变成一张**弯曲的曲面**——"投影到曲面"
没有公式可解,碗也不再是碗(会有沟壑和多个洼地),于是只剩下山这一条路。
这就是 2.2 那句括号的完整版。

三个 demo 到此串成了一条线:

> **cos 量影子(点积 demo)→ 神经元打分是投影(点积 demo 第四节)→
> 最小二乘是把观测投影到模型空间(本 demo)。同一只影子,三个尺度。**

## 四、试试这几种情形

- **制造一个离群点**:把某个红点拖到很远,看它的黄色正方形以平方速度膨胀、
  直线被狠狠拽过去——平方"大错重罚"的直观代价。这也是为什么对脏数据
  常改用对离群点更宽容的损失(如绝对值/Huber),那属于"稳健回归"的话题;
- **手动挑战最优**:关掉绿色参考线,拖直线把 SSE 压到最小,再打开参考线
  看你离 k*、b* 有多近——你的眼睛其实很擅长当优化器;
- **只留两个点**(双击删掉其余):两点确定一条直线,SSE 可以压到 0——
  参数个数 = 数据个数时"完美拟合"毫无难度,也毫无泛化可言,
  这是「什么是神经网络」里"死记硬背 vs 学规律"的最小版本;
- **把点排成竖线**:x 全相同,面板 k* 显示"—"——竖直方向的"直线"
  斜率无穷大,闭式解的分母 Σ(x−x̄)² = 0,数学上无解;
- **从很离谱的直线开始「一键拟合」**:不管起点多差,碗只有一个底,
  梯度下降总能滚到同一个终点——二次碗的世界里没有"局部最优"陷阱,
  这是线性模型比深度网络幸福的地方。

## 五、它通向哪里

- **均方误差(MSE)**:把 SSE 除以 n 就是 MSE——神经网络做回归任务时的
  标准损失函数,和 what-is-nn 里的交叉熵是姊妹;
- **线性探针**:研究大模型内部表征的常用工具——拿隐藏层向量对目标做一次
  线性回归,看"这个信息是否被编码了"(what-is-nn README 里"从 64 个
  隐藏值反推图片平移量"用的正是它);
- **多元与非线性**:把"一条直线"换成"多个特征的加权和"乃至多项式,
  闭式解与投影的几何一字不改,只是模型平面升维;
- **下一个 demo(微分 · 导数)**:2.2 节"摸一摸脚下坡度"里那个"坡度",
  就是导数——割线逼近切线的故事,正好接在后面。
