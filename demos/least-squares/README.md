# 回归 · 最小二乘 vs 最小绝对偏差

> 同一堆散点,两种"最好的直线":绿色是让**残差平方和**最小的最小二乘(L2),
> 紫色是让**残差绝对值和**最小的最小绝对偏差(L1)。拖动散点,看两条线怎么
> 分道扬镳——尤其把一个点拖成离群点时,绿线被狠狠拽走,紫线纹丝不动。
> 顺便回收「点积」demo 埋下的伏笔:最小二乘的解,就是一次投影。

## 如何使用

1. 打开本目录的 [index.html](https://wanghaoming.github.io/math4llm/demos/least-squares/index.html)(或从合集[首页](https://wanghaoming.github.io/math4llm/)进入)。
2. 画布上有一批**红色散点**和一条**蓝色直线**(由两个蓝色端点定义):
   - **拖动红点**改变数据;**点击空白处**加新点(最多 15 个);**双击红点**删除它;
   - **拖动两个蓝色端点**手动调整直线,右侧面板实时显示斜率 k、截距 b,以及这条线的
     两种代价:残差平方和 **SSE** 和残差绝对值和 **SAE**。
3. 两条参考线(默认都开):
   - **绿色 L2 线**:让 SSE 最小的最小二乘直线,由闭式解算出;
   - **紫色 L1 线**:让 SAE 最小的最小绝对偏差直线,由暴力枚举点对精确算出。
4. 四个开关:
   - **显示残差竖线**:每个点到蓝线的竖直虚线,长度 = 这个点的"误差";**所有竖线
     的总长度 = SAE**;
   - **显示残差的平方**:以残差为边长的黄色正方形——**面积**就是这个点对 SSE 的贡献,
     **所有黄色面积之和 = SSE**;
   - **显示 L2 最小二乘线 / L1 最小绝对偏差线**:分别开关绿、紫两条最优参考线。
5. 三个按钮(拟合是**一步一步**走的,用不带导数的**试探法**,每点一次只走一步):
   - **L2 · 最小二乘试探一步**:把蓝线朝四个方向各试探一点(斜率 k 加/减、
     截距 b 加/减),算一下 SSE,保留最小的那个方向;反复点它,蓝线一步步逼近绿线;
   - **L1 · 绝对值误差试探一步**:同样的试探法,只是改看 SAE 谁最小,逼近紫线;
   - **重置数据**:恢复初始散点与直线。
   - 每走一步,面板上方的**「试探法」框**都会显示细节:四个方向各自的误差、
     选了哪个方向、k 和 b 怎么变、走了多大步、当前试探步长、误差从多少降到多少、
     离最优还差多远。四个方向都没更好时会**把试探步长减半**,试得更精细。
     拖动散点或直线端点会清空步数、重新开始。

先手动拖蓝线,试着把 SSE 或 SAE 压到最小,再和绿线、紫线比一比;然后交给按钮,
一步步看它怎么凭感觉试探着挪。最后**把一个红点拖到很远**,盯着绿线和紫线的不同反应——
这就是本 demo 的高潮。

## 〇、先说结论

> **"最好的直线"取决于你怎么惩罚误差。**
> 把误差**平方**再求和 → 最小二乘(L2):有整齐的闭式解、几何上是一次投影,
> 但对离群点极其敏感(大错被平方放大)。
> 把误差取**绝对值**再求和 → 最小绝对偏差(L1):没有闭式解(本质是线性规划),
> 但对离群点**稳健**——一个坏点也拽不动它。
> L2 之于均值,正如 L1 之于中位数。这个 demo 让你同屏看见两者的分歧。

## 一、问题:一堆散点,哪条直线"最好"?

散点大致沿一条直线分布,但每个点都有噪声。任意画一条直线 y = kx + b,
每个数据点 (xᵢ, yᵢ) 和直线之间就有一段竖直的落差:

$$e_i = y_i - (k x_i + b) \qquad \text{(第 i 个点的残差:实际值减预测值)}$$

```
        y
        |                * (x2, y2)
        |                .                <- 残差 e2 > 0(点在线上方)
        |             ___.____
        |     ____----
        | ----    .
        |         .                       <- 残差 e1 < 0(点在线下方)
        |         * (x1, y1)
--------+------------------------- x
```

"最好的直线"应该让所有残差整体最小。但"整体"怎么定义?直接把残差加起来
不行——上方的正残差和下方的负残差会互相抵消,一条烂线也能得零分。
所以第一步是**先消掉符号**,再求和。消符号最自然的两种办法:

$$\text{平方:}\ \mathrm{SSE}(k,b) = \sum_i e_i^2 \qquad\qquad \text{绝对值:}\ \mathrm{SAE}(k,b) = \sum_i |e_i|$$

这两种选择,就是本 demo 里绿线和紫线的全部分歧来源。

## 二、平方 vs 绝对值:两种"惩罚哲学"

两种损失都能消掉正负、都随误差单调增大,但它们对"大误差"的态度截然不同。

| | 平方误差 L2(绿) | 绝对值误差 L1(紫) |
| --- | --- | --- |
| 单点代价 | e²(边长为 \|e\| 的正方形**面积**) | \|e\|(残差竖线的**长度**) |
| 对大误差 | **平方放大**:误差翻倍,罚分变 4 倍 | 线性:误差翻倍,罚分才翻倍 |
| 离群点 | 被狠狠拉过去(大错重罚) | 基本无视(每点权重一样) |
| 误差面形状 | 一只光滑的碗 | 一只带棱的漏斗 |
| 精确求解 | 闭式解(几何投影,见第五节) | 枚举点对精确解(第 3.2 节) |
| 对应统计量 | 均值 | 中位数 |

**demo 把这张表画了出来:**

- 勾选"显示残差的平方",黄色正方形的**面积**就是各点的 L2 罚分,SSE = 总面积;
- 勾选"显示残差竖线",竖线的**长度**就是各点的 L1 罚分,SAE = 总长度。

把一个点往远处拖:它的黄色正方形以**平方速度**膨胀(边长翻倍、面积四倍),
于是绿线被这一个点主导、猛地转过去;而它的残差竖线只是**线性**变长,
对 SAE 的影响有限,紫线几乎不动。

> **这就是全部故事:平方把"大错"放大成主角,绝对值让每个点一人一票。
> 前者精确、好算,后者抗噪、稳健。**

## 三、两条线各自怎么算出来的

### 3.1 L2 闭式解:一步到位的公式(面板绿色的 k*、b*)

SSE 是 k、b 的函数——一个开口朝上的光滑"碗"(二次曲面)。碗底(误差最小处)
满足一组恰好把残差"用光"的条件——**第五节**会用纯几何(残差垂直于模型)把它讲透,
这里先直接给出解出来的公式(初中代数就够):

$$k^* = \frac{\sum_i (x_i - \bar{x})(y_i - \bar{y})}{\sum_i (x_i - \bar{x})^2} \qquad b^* = \bar{y} - k^* \bar{x}$$

其中 x̄、ȳ 是均值。两个式子各有一句人话:

- **k\*** 的分子是 x 和 y 的"协同变化量",分母是 x 自身的"分散程度"——
  斜率 = y 随 x 变化的部分,占 x 总变化的比例;
- **b\* 的式子等价于:最优直线必过均值点 (x̄, ȳ)**。拖动散点时盯着绿线,
  它始终穿过数据的"重心"。

分母 Σ(x−x̄)² 是关键:把所有点拖成一条竖线(x 全相同),分母为零,
最优直线不存在——面板显示"—",这不是 bug,是数学在报错。

### 3.2 L1 没有闭式解,但最优线一定"穿过两个点"

绝对值在零点有个尖角,SAE 这个"碗"是一堆平面折出来的、带棱的漏斗,
没有像 3.1 那样一步到位的公式——严格说它是个**线性规划**问题。

但 L1 有个漂亮而实用的性质:

> **让 SAE 最小的直线,一定恰好穿过其中至少两个数据点。**

直觉:如果一条线没贴住任何点,你总能朝"多数点所在的一侧"平移或转动它,
让 SAE 继续下降,直到被某些点"顶住"为止——最优解一定卡在某两个点上。

这条性质让精确求解变得很简单,**demo 就是这么算紫线的**:枚举所有点对
(n≤15,最多 C(15,2)=105 对),每对确定一条候选直线,算出它的 SAE,
取最小的那条。代码见 `bestFitL1()`。x 相同的点对(竖线)跳过;所有点 x
全相同时无解,面板显示"—",和 L2 一样。

(真实世界里点很多时不会枚举点对,而是用**线性规划**或**迭代重加权最小二乘
IRLS** 求解,但对一个十来个点的 demo,枚举点对既精确又直白。)

## 四、更深一层:均值 vs 中位数

把直线退化到最简单的情形能看得更透:如果模型只是"一条水平线 y = c"
(只有截距、没有斜率),那么两种损失分别在求什么?

- **L2:** 让 Σ(yᵢ − c)² 最小的 c,是所有 yᵢ 的**均值**;
- **L1:** 让 Σ|yᵢ − c| 最小的 c,是所有 yᵢ 的**中位数**。

第二个结论很好证:c 每往右挪一点,左边点的 |误差| 增加、右边点的减少,
净变化 = (右边点数 − 左边点数)。只有当两边点数相等——也就是 c 落在
**中位数**上时,再往哪边挪都不划算。

而均值 vs 中位数正是"抗噪"差别的最小版本:一个亿万富翁走进小酒馆,
平均收入暴涨(均值被离群值拽走),中位数几乎不变。L2 继承了均值的敏感,
L1 继承了中位数的稳健——demo 里那条紫线,就是这份稳健在二维上的样子。

> **L2 : 均值 = L1 : 中位数。斜率一加进来,故事不变,只是从"一个数"
> 升级成"一条线"。**

## 五、L2 的几何:最小二乘 = 一次投影

这一节只讲绿线(L2)——因为"投影"这套漂亮的几何,恰恰依赖平方(欧氏距离),
换成绝对值就没有了。这是「点积」demo 结尾埋的伏笔,现在回收。
换一副眼镜看同一个问题:不再把数据看成"平面上的 n 个点",而是把所有观测值
**竖着堆成一个向量**:

$$\mathbf{y} = (y_1, y_2, \ldots, y_n) \qquad \text{(n 维空间里的一个箭头)}$$

直线能产生的预测也是一个 n 维向量,而且只有两个自由度:

$$\hat{\mathbf{y}} = k \cdot (x_1, \ldots, x_n) + b \cdot (1, \ldots, 1)$$

k 和 b 随便取,所有可能的预测 ŷ 张成 n 维空间里的一张**二维平面**
(模型平面)。真实观测 y 因为带噪声,一般**不在**这张平面上。
于是"找最好的直线"变成了一个纯几何问题:

> **在模型平面上,找离 y 最近的那个点。**

答案你在「点积」demo 里已经见过:**把 y 垂直投影到平面上**。

```
              * y    (观测向量, n 维空间中一点)
             /.
            / .
           /  .      <- 残差向量 e = y - yhat, 垂直于平面
          /   .
         /    .
   -----+-----+----------------
        O     yhat   (y 在模型平面上的投影 = 最小二乘的预测)
```

"最近" = 残差向量 e 最短,而 SSE 恰好就是 |e|²(平方和 = 长度的平方,
「点积」demo 3.4 的副产品 a·a = |a|²)。垂直投影最短,所以:

> **最小二乘的解 = y 在模型平面上的投影;残差向量 ⊥ 模型平面。**

"垂直"翻译成代数,就是两个点积为零(e 和平面的两个方向都零贡献——
「点积」demo 2.2 那个"垂直 = 互不贡献"):

$$\mathbf{e} \cdot (x_1, \ldots, x_n) = 0 \qquad \mathbf{e} \cdot (1, \ldots, 1) = 0$$

把这两个点积展开,得到的正是 3.1 节的闭式解——教科书管它叫**正规方程**。
(为什么 L1 没有这套?因为 SAE = Σ|eᵢ| 不是欧氏长度的平方,"最近"不再是
垂直投影,勾股定理失效,漂亮的正交几何就散了。)

## 六、一步一步"凭感觉"拟合:两个「试探法」按钮

绿线、紫线是**一步到位**算出来的最优解。但换个处境:假如你**不知道**第三节那些
公式,只会"给一条直线、算出它的误差",怎么找到最好的线?最土、也最万能的办法
就是——**试**。demo 把这个笨办法交给你,每点一次按钮走一步。

### 6.1 试探法(爬山)怎么走

站在当前这条蓝线上,朝四个方向各试探一小步:

- **斜率调大 / 调小**(把线绕着转一点点)——对应 k+δ、k−δ;
- **截距调高 / 调低**(把线整体上下平移一点点)——对应 b+δ、b−δ。

每个方向都算一下误差(L2 按钮看 SSE,L1 按钮看 SAE),**四个里挑误差最小的那个,
就往那儿挪**。如果四个方向都没能让误差更小(说明快到谷底、步子迈过头了),
就**把试探步长 δ 减半**,下一次试得更精细。如此反复,直到 δ 趋近于 0,
蓝线停在绿线 / 紫线上。

面板的「试探法」框把每一步都摊开给你看:四个方向各自的误差、这步选了谁、
k 和 b 怎么变、走了多大步、当前试探步长、误差从多少降到多少、离最优还差多远。
你能一眼看到它怎么"摸黑下山"。

### 6.2 笨,但通用——因为它不需要导数

关键是:整个过程只用到两件事——**会算误差、会比大小**。没有公式、没有求导。
正因如此,它对平方(SSE)和绝对值(SAE)**一视同仁**:同一套按钮、同一套逻辑,
换个误差就能用。这就是"笨办法"的好处——无脑、通用,什么损失都能试。

代价也很明显:每一步都在**盲试**四个方向,还常常空试(减半那几步其实原地没动);
越接近谷底越慢,要几十步才彻底收敛。你自己点上十几下就会冒出一个念头:

> **能不能不用盲试,就直接知道"往哪个方向下坡最快、该迈多大步"?**

能。那需要一个新工具——**导数**。导数能算出误差在当前直线处的"下坡方向和陡峭
程度",于是一步就朝最优方向精准迈进,不用四处试探。这就是**梯度下降**。

### 6.3 这正是下一节的事

**下一个 demo(微分 · 导数)会专门讲导数,然后用梯度下降回来取代这里的试探法**
——同一只碗,聪明地下山:方向不靠试、靠算,步子更少、更准。

顺带一提:真实神经网络的训练用的就是这个"聪明下山",只是把 2 个参数(k、b)
换成成千上万个。**线性回归就是最简单的"神经网络"**(一个神经元、恒等激活、
均方误差损失);本节的试探法是它的"无导数平民版",先帮你把"一步步逼近最优"
的直觉建起来,等导数到位,机器就换上更快的鞋。

而"最优到底长什么样",第五节已经用几何回答过了。三个 demo 到此串成一条线:

> **cos 量影子(点积 demo)→ 神经元打分是投影(点积 demo 第四节)→
> 最小二乘是把观测投影到模型空间(本 demo)。同一只影子,三个尺度。**

## 七、试试这几种情形

- **制造一个离群点(本 demo 的招牌)**:把某个红点拖到很远,盯着两条线——
  绿线(L2)被它以平方速度膨胀的黄色正方形拽着猛转,紫线(L1)几乎不动。
  这是"平方大错重罚 vs 绝对值一人一票"最直观的对照,也是脏数据场景改用
  **稳健回归**(L1 / Huber)的全部理由;
- **一步步摸黑下山**:从歪的蓝线开始,反复点「L2 试探一步」,盯着面板——四个
  方向的误差此消彼长、步长 δ 在"空试"后一次次减半,越接近谷底走得越慢;再用
  「L1 试探一步」跑一遍,对比两者试探的路径和步数,亲身体会笨办法的又通用又慢;
- **先制造离群点,再让两个按钮各跑到底**:拖出一个坏点后分别按到收敛,
  蓝线停在绿线时被坏点拽偏、停在紫线时几乎没动——用"拟合的终点"再确认一次
  两种损失的性格;
- **手动挑战最优**:关掉绿、紫两条参考线,拖蓝线把 SSE 压到最小,再打开
  绿线看差多少;换个目标把 SAE 压到最小,再打开紫线——你会发现让 SSE 最小
  和让 SAE 最小往往**不是同一条线**;
- **只留两个点**(双击删掉其余):两点确定一条直线,SSE 和 SAE 同时压到 0——
  绿线紫线重合,谁都完美,也谁都没学到规律;
- **让数据很"干净"**:噪声很小、没有离群点时,绿线和紫线几乎贴在一起——
  两种损失只在有坏点时才吵架,数据老实时它们意见一致;
- **把点排成竖线**:x 全相同,面板 k* 和 L1 的 k 都显示"—"——竖直"直线"
  斜率无穷大,两种方法同时失效;
- **从很离谱的直线开始试探**:不管起点多差,SSE 的碗只有一个底,试探法总能
  一路挪到绿线——二次碗里没有"局部最优"陷阱,连笨办法都不会走岔,这是线性 L2
  模型比深度网络幸福的地方。

## 八、它通向哪里

- **均方误差(MSE)**:把 SSE 除以 n 就是 MSE——神经网络做回归任务时的
  标准损失函数,和 what-is-nn 里的交叉熵是姊妹;
- **稳健回归**:L1(最小绝对偏差)、Huber 损失等专门为"有离群点/脏数据"
  设计,本 demo 的紫线就是最朴素的一员;深度学习里同样常用 L1/Huber 来
  压低异常样本的影响力;
- **线性探针**:研究大模型内部表征的常用工具——拿隐藏层向量对目标做一次
  线性回归,看"这个信息是否被编码了";
