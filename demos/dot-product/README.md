# 向量加法 · 点积

> 拖动两个向量,直观理解「加法」与「点积」到底在做什么。

## 如何使用

1. 打开本目录的 `index.html`(或从合集首页进入)。
2. 画布上有两个箭头:**a**(红) 和 **b**(蓝)。
3. **拖动箭头端点**改变向量;右侧面板实时显示坐标、点积、夹角、投影。
4. 两个开关:
  - **显示平行四边形 a + b**:看向量加法的几何构造(绿色)。
  - **显示投影**:把 a 投影到 b 上(黄色箭头),这就是点积的几何意义。白色虚线是 a 端点到 b 的「垂线」,垂足就是投影的终点。

试试这几种情形,盯着面板里「点积 a · b」的符号和「夹角 θ」:

- 让 a、b **同向** → θ→0°,点积最大(正)。
- 让它们 **垂直** → θ=90°,点积 = 0(黄色投影箭头缩成一个点)。
- 让它们 **反向** → θ→180°,点积为负(黄色箭头指向 b 的反方向)。
- 固定方向、**只把 b 拉长** → 点积随 |b| 成比例变大,但夹角不变。

## 一、点积怎么算(计算原理)

点积把**两个向量**吃进去,吐出**一个数**(标量)。它有两个等价的定义:

```
代数定义:  a · b = a_x·b_x + a_y·b_y          （逐分量相乘再相加）
几何定义:  a · b = |a| · |b| · cos θ          （长度 × 长度 × 夹角余弦）
```

demo 面板里那行公式 $a \cdot b = |a||b|\cos\theta$ 就是几何定义;而 `sketch.js` 里
真正算出来的是代数定义 `a.x*b.x + a.y*b.y`。两者**永远给出同一个数**——这不是巧合,
而是点积的核心。

### 为什么这两个定义相等?

关键在于:代数定义天然满足**分配律**和**线性**,而几何定义天然满足**旋转不变性**。
用余弦定理可以直接把两者对上。设 a、b 夹角为 θ,考虑向量 a−b 构成的三角形:

```
|a − b|²  = |a|² + |b|² − 2|a||b|cos θ        ← 余弦定理(几何)
|a − b|²  = (aₓ−bₓ)² + (a_y−b_y)²             ← 坐标展开(代数)
          = |a|² + |b|² − 2(aₓbₓ + a_y b_y)
```

两式右边相减,`|a|²` 和 `|b|²` 抵消,立刻得到:

```
aₓbₓ + a_y b_y  =  |a||b|cos θ
```

左边是代数定义,右边是几何定义——它们是同一个量。**这就是点积计算原理的全部秘密**:
一个便于用坐标快速计算(代数),一个便于理解它的含义(几何)。

### 从代数定义反解出所有几何量

有了 `a · b` 这一个数,几何信息就都能算出来(面板里做的正是这些):

```
长度:    |a| = √(a · a)                       （自己点自己 = 长度平方）
夹角:    cos θ = (a · b) / (|a||b|)            （反解出 θ）
标量投影: a 在 b 上的投影长度 = (a · b) / |b|
```

面板里的「a 在 b 上的投影」这一行,算的就是 `(a·b)/|b|`;拖动时观察它和黄色箭头长度的对应关系。

## 二、几何意义

### 1. 向量加法 = 首尾相接

向量 `a + b` 的意思是:**先沿 a 走,再沿 b 走**,最终到达的位置。

把 b 平移到 a 的末端(或反过来),从原点指向终点的那个箭头就是 `a + b`。两条平移得到的虚线和原来的 a、b 围成一个**平行四边形**,`a + b` 正是它的对角线。这就是"平行四边形法则"。

> 物理直觉:两个力同时作用在一个物体上,合力就是它们的向量和;一个人在匀速行驶的船上行走,对地的速度是"对船速度 + 船对地速度"。

### 2. 点积 = 一个向量在另一个向量方向上"有多少"

几何定义最能说明点积"是什么":

- `|a| · cos θ` 是 **a 投影到 b 方向上的长度**(标量投影,可正可负)。
- 所以 **a · b = |b| × (a 在 b 上的投影长度)**。

换句话说,点积衡量的是 **a 与 b "方向有多一致"**,再按 b 的长度放大。demo 里的黄色箭头就是
a 投影到 b 上的**投影向量**;它的长度乘以 |b| 就得到点积。这也解释了 demo 面板下方那句提示:
「点积 = |b| × (a 投影到 b 的长度)」。

一个重要直觉:点积对**两个向量都对称**(a·b = b·a),所以"谁投影到谁"结果一样;
但它同时受**方向一致程度**(cos θ)和**两者长度**(|a|、|b|)三者影响——拖动 demo 时,
方向变会改变符号,拉长任一向量会等比例放大数值。

### 3. 为什么符号有意义

`cos θ` 决定了点积的符号,这正是它最有用的地方:


| 两向量关系 | θ    | cos θ | a · b | 含义   |
| ----- | ---- | ----- | ----- | ---- |
| 同向    | 0°   | 1     | 最大正值  | 完全一致 |
| 锐角    | <90° | 正     | 正     | 大致同向 |
| 垂直    | 90°  | 0     | **0** | 互不相关 |
| 钝角    | >90° | 负     | 负     | 大致相反 |
| 反向    | 180° | −1    | 最小负值  | 完全相反 |


**点积 = 0 ⟺ 两向量垂直(正交)**——这是判断正交最简单的方法,也是投影、最小二乘、傅里叶分解等一大批方法的基石。

> 物理直觉:**功 = 力 · 位移**(W = F · d = |F||d|cos θ)。只有力沿位移方向的分量才做功;力与运动垂直时(如向心力)不做功——这正是点积为 0 的情形。

### 4. 它通向哪里

- **投影**:把 a 投影到 b,用的就是点积 → 后续「最小二乘」demo 的核心。
- **长度与夹角**:`|a|² = a · a`;`cos θ = (a·b)/(|a||b|)` 可反解夹角。
- **正交性**:点积为 0 是"互不干扰"的代数表达,贯穿线性代数。

## 三、为什么点积对神经网络的实现最重要

如果只能保留一个运算来理解现代神经网络,那就是**点积**。原因是:神经网络里几乎所有的
"计算"最后都归结为**大量点积**。下面从 demo 的直觉一路推到深度学习。

### 1. 一个神经元 = 一次点积

单个神经元做的事,就是把输入向量 `x = (x₁, x₂, …, xₙ)` 和它自己的权重向量
`w = (w₁, w₂, …, wₙ)` 做点积,再加偏置、过激活函数:

```
输出 = f( w · x + b ) = f( w₁x₁ + w₂x₂ + … + wₙxₙ + b )
```

这个 `w · x` 就是本 demo 里 `a · b` 的**高维版本**。demo 是 2 维(两个分量相乘相加),
神经元可能是几百上千维,但**计算原理一模一样**:逐分量相乘再求和。

### 2. 点积天然就是"相似度打分"

回忆第二节的几何意义:点积衡量"两个向量方向有多一致"。把权重 `w` 看成神经元想**检测的模式**,
把输入 `x` 看成**当前样本**:

- `x` 与 `w` 方向越一致(demo 里同向)→ 点积越大 → 神经元被"强烈激活"。
- `x` 与 `w` 垂直(互不相关)→ 点积为 0 → 神经元没反应。
- 方向相反 → 点积为负 → 被抑制。

所以**神经元本质是一个"模式匹配器"**:用点积给"输入和我关心的模式有多像"打一个分。
你在 demo 里拖动 a 靠近或远离 b 时看到的分数变化,就是一个神经元在"识别"输入时发生的事。

> 这也是为什么 Transformer 的**注意力机制**核心是 `Q · Kᵀ`:用点积算 query 和每个 key 的
> 相似度,再据此分配注意力权重。词向量、推荐系统里的"语义相似度"也都是点积(或其归一化版本
> 余弦相似度 `cos θ = a·b/(|a||b|)`,正是面板里的那个量)。

### 3. 一整层 = 矩阵乘法 = 一批点积

一层有很多个神经元,每个神经元一组权重。把它们叠成权重矩阵 `W`(每行一个神经元的 `w`),
那么这一层的输出就是:

```
y = W · x        （矩阵 × 向量)
```

**矩阵乘法的每一个元素,都是一次点积**(W 的一行 · x)。把整个 batch 的输入也堆成矩阵,
就变成矩阵 × 矩阵——里面是**成千上万次点积**。于是:

> 训练 / 推理一个神经网络,99% 的浮点运算都花在点积上。

### 4. 为什么这对"实现"至关重要

点积不仅是理论核心,更决定了工程实现:

- **可并行**:每次点积是独立的"乘加"(multiply-accumulate),彼此不依赖,可以同时算。
GPU / TPU 正是为"海量乘加并行"而生;显卡跑 AI 快,本质是点积算得快。
- **一条硬件指令**:现代芯片有专门的 **MAC(乘加)单元**和张量核心,把点积做成底层原语。
cuBLAS、`matmul`、Tensor Core 优化的都是同一件事——把点积做到极致。
- **反向传播也靠它**:梯度沿网络回传时用的还是矩阵乘法(点积的转置形式),训练和推理共享同一套算子。

一句话:**神经网络在数学上是"点积的堆叠",在硬件上是"乘加的并行"**。
理解了本 demo 里 2 维的 `a · b`,你就理解了万亿参数大模型里每一次前向传播最基本的那块积木。

把这个 demo 玩熟:2 维的点积 = 投影 = 相似度打分。把维度从 2 提到几千,把一次点积叠成矩阵乘法,
你面前就是一个神经网络的前向传播。后面的「最小二乘」(把数据向量投影到模型空间)也会顺理成章。
